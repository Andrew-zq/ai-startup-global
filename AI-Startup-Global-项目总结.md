# AI Startup Global 网站项目总结

更新日期：2026 年 7 月 5 日

## 1. 项目定位

AI Startup Global 当前已经从单纯的品牌展示网站，发展为一个包含社区官网、活动平台、会员系统、学习资料和 AI 信息服务的初步产品。

目前网站主要服务四类访问者：

- 游客：浏览公开内容和全部活动介绍。
- 普通用户：注册、登录、查看内容和申请会员。
- 付费会员：报名网站会员活动，并查看会员学习资料。
- 管理员：管理活动、用户、活动参与记录和会员申请。

## 4. 会员系统

会员升级流程如下：

```text
注册 / 登录账户
→ 点击 Global+
→ 进入 Stripe Checkout
→ 开始 30 天免费试用
→ 试用后自动按 CA$29.99 CAD/月续费
→ Stripe Webhook 自动同步 paid 会员状态
```

当前规则：

- 普通用户默认会员等级为 `free`。
- 付费会员等级为 `paid`。
- 用户不再需要先参加活动；登录后即可进入 Stripe Checkout。
- 每个用户仅可使用一次 30 天免费试用。
- 付费会员才能报名网站会员活动。
- 会员状态由 Stripe Webhook 自动同步，管理员可在后台查看和必要时调整。

## 6. 前端结构

当前前端使用：

- HTML
- CSS
- 原生 JavaScript
- Google Fonts
- Supabase JavaScript SDK

前端特征：

- 支持桌面端和移动端响应式布局。
- 支持中文与英文切换。
- 采用轻量静态页面结构。
- 没有使用 React、Vue 或 Next.js。
- 部署在 GitHub Pages。

该架构对于约 150 名注册用户完全够用，加载速度快、部署成本低。随着功能继续增加，建议后期考虑组件化重构，以降低维护复杂度。

## 7. 后端结构

当前没有单独运行的传统后端服务器，采用“静态前端 + Backend as a Service”架构。

主要服务包括：

- Supabase Auth：处理邮箱登录、注册和 Google 登录。
- Supabase Database：保存用户、活动、报名和会员申请。
- Supabase Row Level Security：控制数据库访问权限。
- Google Apps Script：发送 Gmail 会员申请通知。
- GitHub Actions：每天更新 AI Radar 数据。
- GitHub Pages：托管网站前端。

本地的 `server.js` 仅用于在 `http://localhost:5500` 预览静态网站，不承担线上业务后端功能。

## 8. 数据库结构

Supabase 当前包含四张核心业务表。

### 8.1 `profiles`

保存用户资料：

- 用户 ID
- 邮箱
- 姓名
- 普通或付费会员状态
- 已参加活动次数
- 用户或管理员角色
- 创建和更新时间

### 8.2 `events`

保存活动信息：

- 活动 ID
- Luma 或网站会员活动类型
- 中英文标题
- 中英文介绍
- 中英文活动类型
- 中英文地点
- 开始和结束时间
- 报名链接
- 中英文活动总结
- PPT 链接
- 是否公开发布
- 创建者和更新时间

### 8.3 `event_registrations`

保存活动报名记录：

- 活动 ID
- 用户 ID
- 是否实际参加
- 报名时间

同一用户不能重复报名同一场活动。

### 8.4 `membership_requests`

保存会员升级申请：

- 申请用户
- 待审核、批准或拒绝状态
- 审核管理员
- 申请时间
- 审核时间

## 9. 权限与安全

数据库已启用 Supabase Row Level Security。

主要权限规则：

- 所有人可以读取已发布活动。
- 用户只能读取自己的资料、报名记录和会员申请。
- 只有付费会员可以创建网站活动报名记录。
- 只有参加过至少一次活动的用户可以提交会员申请。
- 只有管理员可以管理活动和用户。
- 管理员身份由指定管理员邮箱或数据库角色判断。

Supabase 前端 Publishable Key 可以放在前端代码中；真正的数据保护依赖数据库的 RLS 策略。Service Role Key 不应出现在前端或 GitHub 仓库中。

2. 首页“开始申请”仍然使用邮件链接，没有统一跳转至活动分流或正式申请页面。
3. 项目 README 内容已经过时，仍描述为主要使用浏览器本地存储。

### 中优先级

1. 部分旧版 `localStorage` 原型逻辑仍与 Supabase 逻辑同时存在，建议逐步清理。
2. 当前缺少线上支付、退款和订阅管理。
3. 管理员后台仍属于基础版本，缺少图片上传、草稿、预览和富文本编辑。
4. AI 新闻、论文和工具频道目前仍显示“即将上线”。
