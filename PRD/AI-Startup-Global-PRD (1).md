## 1. 产品概述

### 1.1 一句话定位

AI Startup Global 是温哥华的 AI 创业者与学习者社区,以线下活动为核心体验、月费会员为核心商业模式,网站承担活动分发、会员权益交付和 AI 信息服务。

### 1.2 商业模式总览

| 收入线 | 定价 | 状态 |
|---|---|---|
| 月费会员订阅 | CA$29.99/月,首月免费试用 | V1.2 上线 |
| 收费活动单次票 | CA$29.99/场,会员免费参加 | 短期经 Luma 售票,V1.3 迁站内 |
| 企业赞助 | 按场洽谈 | 运营侧推进,网站提供展示位 |
| 企业内训(B2B) | 半天工作坊 CA$1500–3000/场 | 远期,网站先上介绍页 |

**定价锚点(重要,写入所有营销页):单场票价 = 月费。** 每月参加 1 场以上活动,开会员即净赚(额外获得资料、回放、目录、社群)。

### 1.3 用户角色

| 角色 | 定义 | 核心权限 |
|---|---|---|
| 游客 | 未登录访问者 | 公开内容、活动介绍、公开资料、购买单次活动票 |
| 普通用户(free) | 已注册登录 | 游客权限 + 个人中心 + 满足资格后开通会员试用 |
| 付费会员(paid) | 订阅生效中(含试用)| 免费参加全部活动 + 会员资料与回放 + 会员目录 + 专属社群 |
| 管理员(admin) | 运营者 | 活动、用户、资料、订阅监控、目录审核 |

## 2. 会员权益与订阅

### 2.1 会员权益清单(付费理由)

1. 全部活动免费参加(含单场售价 CA$29.99 的收费活动)。
2. 会员专属学习资料与活动回放录像。
3. 会员目录:查看并展示于社区成员名录(✅ 仅会员可见)。
4. 会员专属微信群(✅ 已定)。
5. 会员活动的站内报名资格。

### 2.2 订阅规则(已确定)

- 月费 CA$29.99,Stripe 订阅,`trial_period_days = 30`,次月起自动扣款。
- 每用户仅一次免费试用(`trial_used` 字段防重复)。
- 扣款失败自动重试,最终失败降级 free,权益即时收回(社群移除为人工操作,见运营手册)。
- 随时可取消,权益保留至当前计费周期结束。

### 2.3 会员开通流程(✅ 已定)

```
注册 → 参加至少 1 次活动并被确认 → 点击「开通会员」
→ RLS 自动校验资格 → Stripe Checkout 绑卡开通试用
→ 试用期即为 paid → Webhook 同步状态
→ 开通成功页展示:社群入群方式 + 会员目录引导
```

管理员人工审批取消,转为事后监控。`membership_requests` 表过渡期保留。

⚠ 待决策:「参加 1 次活动」门槛是否长期保留,建议 3 个月后数据复盘。

### 2.4 试用期激活策略

开通试用后引导完成首个动作(报名最近会员活动 / 解锁热门资料 / 加入社群)。追踪指标:试用期内参加过活动者的续费率 vs 未参加者。

## 3. 活动与售票

### 3.1 活动类型(更新)

| 类型 | 报名与收费 | 面向 |
|---|---|---|
| 免费公开活动 | Luma 免费报名 | 所有人(引流入口) |
| 收费公开活动 | 单次票 CA$29.99;会员免费 | 所有人 |
| 会员专属活动 | 站内报名,仅会员 | 付费会员 |

### 3.2 售票方案(分两步走)

- **短期(即刻可用)**:收费活动使用 Luma 付费票功能;会员通过专属免费码(promo code)0 元报名。代价:平台服务费、售票数据在 Luma。
- **V1.3 迁移站内**:Stripe 一次性支付购票,`ticket_purchases` 表记录,会员身份自动免票。

### 3.3 现场转化动线(运营 + 网站配合)

- 网站提供 `/join` 落地页:权益清单 + 定价锚点话术 + 直达 Stripe Checkout,供活动现场二维码扫码使用。
- 运营侧:每场公开活动结尾 3 分钟会员介绍 + 大屏二维码 + 现场开通限时赠品(如下场会员活动附加权益)。

### 3.4 活动报名流程

与 v1.1 相同:Luma 活动外链报名;会员活动站内报名写入 event_registrations;活动后管理员标记实际参加;同一用户同一活动不可重复报名。

## 4. 功能需求

### 4.1 账户系统(75%)

邮箱注册 + Google 登录(Supabase Auth);新增注册协议勾选(条款 + 隐私政策)。

### 4.2 活动系统(65%)

- events 增加收费属性:免费公开 / 收费公开 / 会员专属;收费活动展示票价与"会员免费"标识。
- 未完成:「最近活动」聚合页;每 3 天自动检查活动更新。

### 4.3 订阅与支付(0%,V1.2 核心)

Stripe Checkout 开通、30 天试用、Webhook 状态同步(Supabase Edge Functions)、取消与降级、用户订阅管理页、管理员订阅监控。

### 4.4 会员目录(新,0%)

- 仅会员可见的成员名录页(✅ 已定)。
- 展示完全自愿:会员在个人中心开启「加入目录」并填写展示资料(姓名/昵称、方向、项目/公司、一句话介绍、可选链接)。
- 默认不展示;可随时下架自己的条目;页面注明信息由成员自愿提供。
- 管理员可下架不当条目。

### 4.5 会员社群(✅ 已定:仅微信群)

- 开通成功页与个人中心向 paid 用户展示入群方式,入群信息仅对 paid 状态可见(RLS 门控)。
- 微信群二维码有 7 天有效期与扫码人数上限,不适合长期挂在页面上。**推荐入群动线**:页面展示管理员个人微信二维码,提示添加时备注注册邮箱;管理员核对 paid 状态后拉入群——顺带完成身份核验,防止入群凭证被截图外传。
- 备选动线:每周更新群二维码并标注"仅限本人使用"(维护成本更高,核验更弱)。
- 会员取消/降级后的清退为人工动作:每月对照管理后台订阅列表清理一次群成员。

### 4.6 学习资料与回放(改造中)

- materials 表管理,public/member 分级;会员资料含**活动回放录像**(每场线下活动录制,剪辑后入库)。
- 会员资料经 Supabase Storage 签名 URL 分发;非会员可见标题与简介,下载口引导开通。
- 回放使内容库随活动持续增值,为订阅制核心资产。
- ⚠ 待决策:现有 4 份资料的公开/会员归类。

### 4.7 企业服务页(新,轻量)

- 「企业内训」介绍页:AI Workflow 系列工作坊介绍、半天 CA$1500–3000 价格区间、过往活动照片、联系方式(域名邮箱 + 表单或 mailto)。
- 「赞助我们」区块:面向本地 AI 工具/云服务/招聘方,展示社区规模与赞助权益,联系方式。
- 纯静态展示页,无后端逻辑,先占位建立 B2B 入口。

### 4.8 内容与信息服务(35%)

AI Radar 已上线;ShouldWeTV 半成品;AI 简报、政策页、新闻/论文/工具频道未开始(排期见第 9 节 V1.4)。

### 4.9 管理员后台(60%)

新增:资料上传与分级管理、订阅监控、目录条目管理、(V1.3)站内售票订单查看。

## 5. 数据模型

### 5.1 profiles

现有字段 + 订阅字段(`stripe_customer_id`、`subscription_status`、`membership_expires_at`、`trial_used`)+ 目录字段(`directory_visible` 布尔、`directory_headline`、`directory_focus`、`directory_project`、`directory_links`)。

用户可自改:目录展示相关字段;严禁自改:role、membership/subscription 相关、trial_used、参与次数(RLS 列级限制)。

### 5.2 events

新增:`pricing_type`(free_public / paid_public / member_only)、`ticket_price`、`luma_promo_note`(过渡期会员免费码说明,仅对 paid 可见)。

### 5.3 event_registrations

不变。唯一约束(活动, 用户)。

### 5.4 membership_requests

过渡期保留,新流程上线后停用归档。

### 5.5 materials

资料 ID、双语标题、关联活动、可见级别(public/member)、类型(ppt/exercise/**video 回放**)、存储路径或公开 URL、上传信息。

### 5.6 ticket_purchases(V1.3 新表)

购票记录:活动 ID、用户 ID(或游客邮箱)、Stripe 支付 ID、金额、状态、购买时间。

## 6. 技术架构

静态前端(GitHub Pages + aistartupglobal.com)+ Supabase(Auth / Database+RLS / Storage / Edge Functions)+ Stripe(订阅 + 一次性支付)+ GitHub Actions(AI Radar、每周备份)+ Namecheap(域名,Auto-Renew)。

约束不变:轻量架构不引入框架;Edge Functions 仅承担必须保密的逻辑(Webhook、签名 URL);回放视频体积大,存 Supabase Storage,注意存储用量与带宽成本,超出免费额度时评估 Pro 计划。

---

## 7. 权限与安全需求

- 全表 RLS;前端判断仅用于界面;管理员以 role 字段判定。
- profiles 列级更新限制(见 5.1)。
- paid(含 trialing)才可:会员活动报名、获取会员资料签名 URL、读取入群凭证、读取会员目录。
- Stripe Webhook 校验签名;Secret Key 仅存 Edge Functions 环境变量。
- 会员目录属个人信息展示:必须自愿开启、可随时撤回、页面明示用途(PIPEDA/BC PIPA)。
- 回放视频仅经签名 URL 分发,不落公开仓库。
